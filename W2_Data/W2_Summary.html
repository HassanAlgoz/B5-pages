Week 2 Overview

Week 2 Focus

Week 2 focuses on developing professional data handling skills—from raw data ingestion to clean, well-documented datasets ready for modeling.

Core Philosophy: Good data beats clever models. Build it carefully.

Learning Objectives

By the end of Week 2, you should be able to:

  • Think systematically about data workflows
  • Assess and improve data quality
  • Clean and transform real-world datasets
  • Handle dates, times, and outliers correctly
  • Perform effective exploratory data analysis
  • Produce clean, documented datasets ready for modeling

Canonical Workflow

The 7-Step Process

The week follows a structured workflow (repeat every project):

  1. Load (from raw/cache)
  2. Verify (columns, types, key uniqueness, row counts, missingness)
  3. Clean (types, missing, duplicates, normalization)
  4. Transform (joins, reshape, feature engineering)
  5. Analyze (stats tables, comparisons, bootstraps)
  6. Visualize (charts with titles/labels + export)
  7. Conclude (written summary + caveats + next questions)

Data Separation by Role

Concept: Separate data by purpose to prevent chaos.

Code Pattern:

@dataclass(frozen=True)
class Paths:
    root: Path
    raw: Path      # immutable inputs (never edited)
    cache: Path   # API responses (safe to delete)
    processed: Path  # clean, typed outputs (idempotent)

def make_paths(root: Path) -> Paths:
    data = root / "data"
    return Paths(
        root=root,
        raw=data / "raw",
        cache=data / "cache",
        processed=data / "processed",
    )

Idempotent Outputs

Concept: Rerunning produces the same results (safe to overwrite).

Code Pattern:

# Good: overwrite (idempotent)
df.to_parquet("data/processed/orders.parquet", index=False)

# Bad: append (not idempotent)
df.to_csv("data/processed/orders.csv", mode="a", index=False)  # ❌ duplicates accumulate

Schema-Aware I/O

Concept: Explicit dtypes prevent silent corruption (especially IDs).

Code Pattern:

def read_orders_csv(path: Path) -> pd.DataFrame:
    return pd.read_csv(
        path,
        dtype={"order_id": "string", "user_id": "string"},  # IDs as strings!
        na_values=["", "NA", "N/A", "null", "None"],
        keep_default_na=True,
    )

Parquet for Processed Data

Concept: Parquet preserves types, is faster, and smaller than CSV.

Code Pattern:

def write_parquet(df: pd.DataFrame, path: Path) -> None:
    path.parent.mkdir(parents=True, exist_ok=True)
    df.to_parquet(path, index=False)  # Preserves dtypes automatically

def read_parquet(path: Path) -> pd.DataFrame:
    return pd.read_parquet(path)  # Types restored automatically

Day 2: Quality & Cleaning

Day 2 Goal

Make your dataset trustworthy by adding checks and cleaning for missingness, duplicates, and text normalization.

Fail Fast Validation

Concept: Validate assumptions early with clear errors.

Code Pattern:

def require_columns(df: pd.DataFrame, cols: list[str]) -> None:
    missing = [c for c in cols if c not in df.columns]
    assert not missing, f"Missing columns: {missing}"

def assert_unique_key(df: pd.DataFrame, key: str, *, allow_na: bool = False) -> None:
    if not allow_na:
        assert df[key].notna().all(), f"{key} contains NA"
    dup = df[key].duplicated(keep=False) & df[key].notna()
    assert not dup.any(), f"{key} not unique; {dup.sum()} duplicate rows"

Missingness Strategy

Concept: Measure first, avoid blanket drops, default to flagging.

Code Pattern:

def missingness_report(df: pd.DataFrame) -> pd.DataFrame:
    n = len(df)
    return (
        df.isna().sum()
          .rename("n_missing")
          .to_frame()
          .assign(p_missing=lambda t: t["n_missing"] / n)
          .sort_values("p_missing", ascending=False)
    )

def add_missing_flags(df: pd.DataFrame, cols: list[str]) -> pd.DataFrame:
    out = df.copy()
    for c in cols:
        out[f"{c}__isna"] = out[c].isna()  # Keep missingness visible
    return out

Text Normalization

Concept: Normalize categories to prevent fake groups.

Code Pattern:

import re
_ws = re.compile(r"\s+")

def normalize_text(s: pd.Series) -> pd.Series:
    return (
        s.astype("string")
         .str.strip()
         .str.casefold()  # Robust lowercase
         .str.replace(_ws, " ", regex=True)  # Collapse whitespace
    )

def apply_mapping(s: pd.Series, mapping: dict[str, str]) -> pd.Series:
    return s.map(lambda x: mapping.get(x, x))  # Controlled synonyms

Day 3: Datetimes & Joins

Day 3 Goal

Build an analytics-ready table by parsing time correctly, handling outliers safely, and joining without corrupting data.

Datetime Parsing

Concept: Parse safely with error handling and timezone awareness.

Code Pattern:

def parse_datetime(df: pd.DataFrame, col: str, *, utc: bool = True) -> pd.DataFrame:
    dt = pd.to_datetime(df[col], errors="coerce", utc=utc)  # Invalid → missing
    return df.assign(**{col: dt})

def add_time_parts(df: pd.DataFrame, ts_col: str) -> pd.DataFrame:
    ts = df[ts_col]
    return df.assign(
        date=ts.dt.date,
        year=ts.dt.year,
        month=ts.dt.to_period("M").astype("string"),
        dow=ts.dt.day_name(),
        hour=ts.dt.hour,
    )

Outlier Management

Concept: Identify outliers, flag them, cap for visualization (don’t delete).

Code Pattern:

def iqr_bounds(s: pd.Series, k: float = 1.5) -> tuple[float, float]:
    q1 = s.quantile(0.25)
    q3 = s.quantile(0.75)
    iqr = q3 - q1
    return float(q1 - k * iqr), float(q3 + k * iqr)

def winsorize(s: pd.Series, lo: float = 0.01, hi: float = 0.99) -> pd.Series:
    a, b = s.quantile(lo), s.quantile(hi)
    return s.clip(lower=a, upper=b)  # Cap for charts, keep rows

Safe Joins

Concept: Validate cardinality, check row counts, prevent explosions.

Code Pattern:

def safe_left_join(
    left: pd.DataFrame,
    right: pd.DataFrame,
    on: str | list[str],
    *,
    validate: str,  # "many_to_one", "one_to_one", etc.
    suffixes: tuple[str, str] = ("", "_r"),
) -> pd.DataFrame:
    return left.merge(
        right,
        how="left",
        on=on,
        validate=validate,  # Prevents join explosions
        suffixes=suffixes,
    )

# Usage: orders (many) → users (one)
assert_unique_key(users, "user_id")  # Pre-check
joined = safe_left_join(orders, users, on="user_id", validate="many_to_one")
assert len(joined) == len(orders), "Row count changed (join explosion?)"

Bootstrap Uncertainty

Concept: Quantify uncertainty with simulation-based confidence intervals.

Code Pattern:

import numpy as np
import pandas as pd

def bootstrap_diff_means(a: pd.Series, b: pd.Series, *, n_boot: int = 2000, seed: int = 0) -> dict:
    rng = np.random.default_rng(seed)  # Fixed seed for reproducibility
    a = pd.to_numeric(a, errors="coerce").dropna().to_numpy()
    b = pd.to_numeric(b, errors="coerce").dropna().to_numpy()
    
    diffs = []
    for _ in range(n_boot):
        sa = rng.choice(a, size=len(a), replace=True)  # Resample with replacement
        sb = rng.choice(b, size=len(b), replace=True)
        diffs.append(sa.mean() - sb.mean())
    diffs = np.array(diffs)
    
    return {
        "diff_mean": float(a.mean() - b.mean()),
        "ci_low": float(np.quantile(diffs, 0.025)),
        "ci_high": float(np.quantile(diffs, 0.975)),
    }

Day 5: Pipeline & Handoff

Day 5 Goal

Package Week 2 work as a job-ready handoff: reproducible ETL → processed datasets → EDA notebook → exported figures → written summary.

ETL Structure

Concept: Separate Extract, Transform, Load with logging and validation.

Code Pattern:

@dataclass(frozen=True)
class ETLConfig:
    root: Path
    raw_orders: Path
    raw_users: Path
    out_analytics: Path
    run_meta: Path

def load_inputs(cfg: ETLConfig) -> tuple[pd.DataFrame, pd.DataFrame]:
    orders = read_orders_csv(cfg.raw_orders)
    users = read_users_csv(cfg.raw_users)
    return orders, users

def transform(orders: pd.DataFrame, users: pd.DataFrame) -> pd.DataFrame:
    require_columns(orders, ["order_id", "user_id", "amount", "created_at", "status"])
    assert_unique_key(users, "user_id")
    
    return (
        orders.pipe(enforce_schema)
              .pipe(parse_datetime, col="created_at", utc=True)
              .pipe(add_time_parts, ts_col="created_at")
              .pipe(lambda d: safe_left_join(d, users, on="user_id", validate="many_to_one"))
    )

def run_etl(cfg: ETLConfig) -> None:
    logging.basicConfig(level=logging.INFO)
    log = logging.getLogger(__name__)
    
    log.info("Loading inputs")
    orders, users = load_inputs(cfg)
    
    log.info("Transforming (orders=%s, users=%s)", len(orders), len(users))
    analytics = transform(orders, users)
    
    log.info("Writing output: %s (%s rows)", cfg.out_analytics, len(analytics))
    write_parquet(analytics, cfg.out_analytics)
    write_run_meta(cfg, rows_out=len(analytics))

Run Metadata

Concept: Track config, counts, and quality stats for reproducibility.

Code Pattern:

import json
from dataclasses import asdict

def write_run_meta(cfg: ETLConfig, *, rows_out: int, missing_ts: int, match_rate: float) -> None:
    meta = {
        "rows_out": rows_out,
        "missing_created_at": missing_ts,
        "country_match_rate": match_rate,
        "config": {k: str(v) for k, v in asdict(cfg).items()},
    }
    cfg.run_meta.parent.mkdir(parents=True, exist_ok=True)
    cfg.run_meta.write_text(json.dumps(meta, indent=2), encoding="utf-8")

Rules That Prevent Pain

Common Patterns

Pure Transform Functions

Concept: Transforms are df -> df, no I/O inside.

Code Pattern:

def clean_orders(df: pd.DataFrame) -> pd.DataFrame:
    return (
        df.pipe(enforce_schema)
          .pipe(parse_datetime, col="created_at", utc=True)
          .assign(status_clean=lambda d: normalize_text(d["status"]))
          .pipe(add_missing_flags, cols=["amount", "quantity"])
    )

# I/O happens outside
orders = read_orders_csv(path)
orders_clean = clean_orders(orders)
write_parquet(orders_clean, output_path)

Pipeline with .pipe()

Concept: Readable top-to-bottom flow.

Code Pattern:

analytics = (
    orders
    .pipe(enforce_schema)
    .pipe(parse_datetime, col="created_at", utc=True)
    .pipe(add_time_parts, ts_col="created_at")
    .pipe(lambda d: safe_left_join(d, users, on="user_id", validate="many_to_one"))
    .assign(amount_winsor=lambda d: winsorize(d["amount"]))
)

Common Pitfalls

Dtype Inference Lies

Concept: Pandas guesses wrong, leading zeros lost forever.

Code Pattern:

# ❌ Bad: pandas infers int64
df = pd.read_csv("orders.csv")
# order_id "0007" becomes 7 (unrecoverable)

# ✅ Good: explicit dtype
df = pd.read_csv("orders.csv", dtype={"order_id": "string"})
# order_id "0007" stays "0007"

Join Explosions

Concept: Keys not unique → row count multiplies.

Code Pattern:

# ❌ Bad: no validation
joined = orders.merge(users, on="user_id", how="left")
# If users has duplicates, joined has more rows than orders!

# ✅ Good: validate first
assert_unique_key(users, "user_id")  # Pre-check
joined = safe_left_join(orders, users, on="user_id", validate="many_to_one")
assert len(joined) == len(orders), "Row count changed"

Blanket .dropna()

Concept: Silently deletes data and biases results.

Code Pattern:

# ❌ Bad: loses information
df = df.dropna()  # Deletes rows, may bias analysis

# ✅ Good: flag missingness
df = add_missing_flags(df, cols=["amount", "quantity"])
# Now you can analyze: "Are missing amounts more common in one country?"
